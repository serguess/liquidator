Прогон $ARGUMENTS статей подряд через полный конвейер.

## Источник тем
Прочитай `data/queue.json`. Структура:
```json
[
  {"category": "fiz", "topic": "...", "status": "pending"},
  {"category": "yur", "topic": "...", "status": "pending"}
]
```
Если файла нет или очередь пуста - сообщи и остановись.

## Алгоритм

1. Возьми первые N записей со `status: "pending"`.
2. Для каждой статьи `i` от 1 до N:
   - Сформируй параметры батча:
     - `batch_position = i`
     - `batch_total = N`
     - `prev_article_summary` - см. раздел «Антинакопление контекста» ниже. При `i = 1` отсутствует.
     - `prev_article_slug` - slug статьи `i-1`. При `i = 1` отсутствует.
   - Запусти конвейер (как в /write-article), передав эти параметры в prompt агентов.
   - По завершении пометь запись `status: "done"` (или `"failed"` с причиной).
   - Запиши результат в `data/batch_log.json` (append).
3. После всех - сводный отчёт:
   - Успешно: X
   - С ошибками: Y (по каждой - slug и причина)
   - Среднее время на статью
   - Общее время
   - Среднее по метрикам text.ru: AI-detector, уникальность, заспамленность (из meta.json)

## Антинакопление контекста (важно)

**Правила оркестратора:**

1. **Не читать `draft.md` полностью между статьями.** После завершения статьи `i` оркестратор сохраняет в локальную переменную (или временный файл `drafts/_batch_state.json`) только сжатый `prev_article_summary`:
   ```json
   {
     "slug": "kak-otmenit-sudebnyj-prikaz",
     "intro_first_sentence": "Сразу к сути. ...",
     "h2_list": ["Откуда взялся приказ", "Что вам даёт отмена", "..."],
     "final_paragraph_formula": "перечисление сценариев действий",
     "main_keyword": "судебный приказ"
   }
   ```
   Это 5-6 строк, а не весь текст статьи. Получено из `meta.json` + первого/последнего абзаца draft.md.

2. **Сразу освобождать контекст оркестратора** от полных HTML/MD-файлов предыдущих статей. Если нужно проверить что-то в прошлой статье - читать заново из файла, не держать в памяти.

3. **Каждый запуск агента 4 - новый Task tool invocation.** Агенты получают чистый контекст с нуля, в их prompt - только параметры из этого `batch-run.md` (включая `prev_article_summary`) и пути к файлам, которые они сами прочитают.

4. **При возврате на агента 4** (по `passed: false` от агентов 5/6) контекст итерации тоже изолирован: новый Task с тем же `batch_position`, `prev_article_summary` и плюс `previous_iteration_issues` - см. write-article.md.

## Ограничения

- Между статьями паузу не делать, но если три подряд упали на одном агенте - остановиться и сообщить.
- В режиме draft (по умолчанию) ничего не пушить в git.
- Не запускать параллельно - конвейер строго последовательный (агенты пишут в одни и те же data-файлы).
- Не накапливать содержимое предыдущих драфтов в контексте оркестратора (см. раздел выше).
