Запусти агента 8 (критик) для анализа последних статей и улучшения базы знаний писателя.

Аргументы: $ARGUMENTS (число, по умолчанию 5)

## Что делать

1. Если аргумент не передан - использовать N=5.
2. Найти N последних `article.html` (по mtime) в `articles/{cat}/` и `drafts/{slug}/`. Если общее число < 3 - сообщить «слишком мало материала, нужно минимум 3 статьи» и остановиться.
3. Запустить агента `8-critic`. Передать список slug-ов и пути к article.html.
4. После работы агента - вывести краткий отчёт:
   - сколько статей проанализировано;
   - сколько правил добавлено в cheatsheet;
   - сколько новых regex в ai_markers_check.py;
   - путь к полному отчёту (`data/critic_report_*.md`).

## Когда полезно

- После прогона батча из 5-10 статей подряд: критик соберёт повторяющиеся проблемы, которые quality_gate не видит на уровне одной статьи.
- После правок в стайл-гайдах: проверить, какой эффект дали изменения, и дополнить.
- Раз в неделю как профилактика дрифта стиля.

## Что НЕ делает

- Не правит сами статьи (для этого есть quality_gate + write-article итерации).
- Не запускает text.ru API (это уже сделал агент 6 для каждой статьи).
- Не публикует ничего на сайт.
