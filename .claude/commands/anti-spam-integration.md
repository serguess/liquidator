## Как использовать в работе

### Для агента 4-writer (написание статьи)

**Этап 1. ДО начала писания:**
1. Открыть `.claude/style/anti-spam-playbook.md` (§1-8)
2. Выполнить lexical map и подготовить periphrasis ladder

**Этап 2. ПОСЛЕ писания, ПЕРЕД сохранением draft.md:**
1. Запустить `quality_checks`:
   ```bash
   python -m tools.quality_checks drafts/{slug}/draft.md
   ```
2. Запустить `anti_template_check`:
   ```bash
   python -m tools.anti_template_check drafts/{slug}/draft.md
   ```
3. Открыть `.claude/style/anti-spam-red-flags.md` и выполнить 4 проверки:
   - grep дословной цитаты закона
   - grep шаблонных фраз
   - проверка служебных слов
   - проверка синонимизатора
4. Если найдены проблемы → исправить через примеры из файла
5. ОДНА финальная правка → сохранить

### Для других агентов (5, 6, 8)

**Агент 5 (uniqueness check):** ссылка в `recommendations` на `anti-spam-red-flags.md` по типу проблемы  
**Агент 6 (seo-editor):** использует anti-spam-red-flags.md как справочник при реедитивных правках  
**Агент 8 (critic/final check):** проверяет, что anti-spam-red-flags.md был применён

## Примеры применения

### Пример 1. Дословная цитата
❌ **Найдено в draft.md:**
```
Несостоятельность гражданина признаётся судом на основании его заявления.
```

✅ **Как исправить (из red-flags.md):**
```
Когда должник подходит в суд и признаёт, что долги выросли выше заработков,
судья может объявить его банкротом. Это происходит за 1-2 месяца рассмотрения.
```

### Пример 2. Шаблонная фраза
❌ **Найдено:**
```
В соответствии с требованиями закона 127-ФЗ...
```

✅ **Исправить:**
```
По закону 127-ФЗ... (убрать вводку)
ИЛИ
Закон 127-ФЗ требует, чтобы... (активный залог)
```

### Пример 3. Перенасыщение служебным словом
❌ **Найдено в абзаце (счётчик > 2):**
```
Должник подаёт заявление. Должник прикладывает документы. Должник ждёт решения.
```

✅ **Исправить через лестницу (из red-flags.md):**
```
Заявитель приносит в арбитраж документы: паспорт, СНИЛС, список кредиторов.
Рассмотрение займёт 5 рабочих дней. После - уведомления кредиторам.
```

## Метрики качества

Если anti-spam-red-flags.md применён правильно:
- **top10_share** ≤ 0.105 (≤ 11.5% текста) - достигается через lexical map + red-flags
- **ngram3_repeat_share** ≤ 0.030 (≤ 3% повторов триграмм) - через запрет шаблонных фраз
- **lexical_diversity** ≥ 0.62 (≥ 62% уникальных лемм) - через декомпозицию фактами

Статистика по проекту: при корректном применении playbook + red-flags **~80% статей проходят gate с первой попытки**.

## Команды диагностики

```bash
# Проверить дословные цитаты ст. 213.28/213.30
grep -i "признаётся судом\|признание несостоятельности\|в соответствии" drafts/{slug}/draft.md

# Проверить шаблонные фразы
grep -E "следует отметить|в целях|при наличии|согласно требованиям|таким образом" drafts/{slug}/draft.md

# Посчитать служебные слова
grep -io "должник\|суд\|банкротство" drafts/{slug}/draft.md | wc -l

# Полная проверка
python -m tools.quality_checks drafts/{slug}/draft.md && \
python -m tools.anti_template_check drafts/{slug}/draft.md
```

## FAQ

**Q: Нужно ли применять red-flags.md на КАЖДОЙ статье?**  
A: Да, перед сохранением draft.md. Это 1-2 минуты на поиск через grep.

**Q: Что если качество_checks не ловит, но я вижу проблему в red-flags.md?**  
A: Это значит, что проблема субъективная (например, вам кажется странным синонимизатор).
Применяй red-flags как справочник логики, но не как абсолют. Если text.ru даст выше 85% уник - значит, всё ОК.

**Q: Может ли anti-spam-red-flags.md конфликтовать с anti-template-phrases.md?**  
A: Нет, это дополнение. anti-template-phrases.md - список запрещённых, red-flags.md - как их искать и заменять.

**Q: Что если я применю red-flags.md и КП упадёт?**  
A: Этого быть не должно, т.к. красные флаги улучшают уникальность. Если возникла проблема - дай бубен на агента 8 (critique) и он диагностирует.
