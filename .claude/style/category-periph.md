# Категорийные перифразы и cap'ы (16 мая 2026)

Каждая категория статей имеет «гипертему» — слово, которое физически тянется через всю статью и в плохих кейсах попадает в топ-1 text.ru с частотой 25-31. У эталонных статей того же домена это слово стоит на 6-8-м месте с частотой 8-12.

Writer обязан **выбрать категорию по `meta.category`** и перед началом писания подгрузить соответствующий блок ниже. Cap'ы — это **бюджет токенов на всю статью**, не на абзац.

---

## mfo (микрозаймы, МФО)

**Гипертема:** «МФО» / «микрофинансовая организация» / «микрозайм».

**Cap на гипертему: ≤ 12 во всей статье** (плохой кейс 31 вхождение даёт +7 п.п. spam).

**Лестница перифразов** (распределить по статье):
- Заголовок и H1 — «МФО» полным сокращением (1-2 раза, для SEO).
- 1-е упоминание в тексте — «микрофинансовая организация» (полное название).
- 2-е и далее — варьировать:
  - «займодавец»
  - «такая компания» / «эта организация»
  - «кредитор» (когда контекст однозначный)
  - «контора» (разговорно, для эмоциональных кусков)
  - конкретное имя из research.json («Быстроденьги», «Займер»)
  - «он» / «она» (когда займодавец субъект предложения)

**Запрещено:** «МФО» × 3 в одном абзаце; «МФО МФО МФО» в перечислении.

**Сопутствующие токены:**
- «микрозайм» / «микрокредит» ≤ 6
- «долг» (в этой категории горячий) ≤ 12, варьировать через «остаток», «задолженность», «обязательство»

## zhkh (банкротство по долгам ЖКХ)

**Гипертема:** «ЖКХ» / «коммунал*» / «жилищно-коммунал*».

**Cap (суммарно по всем формам): ≤ 10**.

**Лестница перифразов:**
- 1-е упоминание — «ЖКХ» (SEO).
- 2-е — «коммунальные услуги» / «коммунальные платежи».
- 3-е — «оплата квартиры», «оплата жилья».
- 4-е — конкретный поставщик: «УК», «ТСЖ», «ресурсник», «энергосбыт», «водоканал».
- 5-е — «эти долги», «такие платежи».

**Сопутствующие токены:**
- «УК ТСЖ» (как пара) ≤ 3 раз
- «единственная квартира» / «единственное жильё» — ≤ 2 раза (горячая legal-формула)

## vzysk (взыскание долга, иски, приставы)

**Гипертемы (две):** «суд» И «взыскан*».

**Cap:** «суд» ≤ 12, «взыскан*» ≤ 8 (плохой кейс — «суд» 18 + «взыскан» 7).

**Лестница перифразов для «суд»:**
- 1-е — «суд».
- 2-е — «арбитраж» / «арбитражный суд» (если ЮЛ).
- 3-е — «районный суд», «мировой суд», «инстанция».
- 4-е — «судья» (когда говорит/решает конкретный человек).
- 5-е — «разбирательство», «дело», «процесс».
- 6-е и далее — «там», «оттуда» (когда подразумевается уже названный суд).

**Лестница для «взыскание»:**
- «взыскание» / «истребовать»
- «возврат» / «вернуть»
- «получить с должника»
- «принудительное исполнение»

**Сопутствующие токены (в этой категории особенно опасные):**
- «руб» = 0 (только «рублей»)
- «X 000 руб» ≤ 2 — суммы по 100/300/500 тысяч встречаются часто, использовать дескриптор «эта сумма» / «такой долг»
- «ГПК» ≤ 3, «ГК» ≤ 3 (плохой кейс 12+10)
- «РФ» ≤ 2

## fiz (банкротство физлица)

**Гипертема:** «банкротств*» / «несостоятельност*».

**Cap (суммарно): ≤ 14**.

**Лестница перифразов:**
- 1-е и H1 — «банкротство» (SEO).
- 2-е — «несостоятельность».
- 3-е — «эта процедура» / «такой путь».
- 4-е — «процесс списания долгов» / «дело о списании».
- 5-е — «обращение в арбитраж».
- 6-е — местоимения / конкретика.

**Сопутствующие токены:**
- «должник» ≤ 8, варьировать через «гражданин», «заявитель», «он», конкретный кейс («женщина 38 лет с двумя кредитами»)
- «процедура» (отдельное слово) ≤ 6
- «127-ФЗ» ≤ 2, «213.28» / «213.30» ≤ 3 суммарно

## yur (банкротство юрлиц, субсидиарка, ликвидация)

**Гипертема:** «компания» / «организация» / «ООО» / «юридическое лицо».

**Cap (суммарно): ≤ 14**.

**Лестница перифразов:**
- «ООО» в авторском тексте — ≤ 3 раза (только когда нужна форма собственности).
- «компания» — основное.
- «организация» — синоним.
- «юридическое лицо» — формальный синоним (1-2 раза).
- «фирма» — разговорно, для живых абзацев.
- Конкретика из research.json («группа компаний X», «застройщик», «торговая сеть»).
- «должник» (если речь о банкротстве компании-должника) — ≤ 6.

**Сопутствующие токены:**
- «руководитель» / «директор» / «первое лицо» / «гендиректор» — варьировать (≤ 6 каждого)
- «КДЛ» (контролирующее должника лицо) — ≤ 3, дальше «контролёр», «он»
- «субсидиарка» / «субсидиарная ответственность» — ≤ 4 суммарно

## news (короткие новости отрасли)

**Длина статьи 4500-6500, поэтому все cap'ы × 0.7.**

- Главная тема статьи (например «закон 369-ФЗ» / «приставы рекордно списали») — ≤ 8
- Авторские вставки — обязательны 2-3 (даже в короткой статье)
- «ст.» ≤ 3, «РФ» ≤ 1, «руб» = 0

## Сквозные cap'ы (одинаковые для ВСЕХ категорий)

| Токен | Cap | Замена |
|---|---|---|
| `ст.` / `ст N` перед числом | ≤ 5 | «статья N» полным словом или «N-я норма» |
| `РФ` (в ГК РФ / ГПК РФ / закон РФ) | ≤ 2 | дропнуть |
| `руб` / `руб.` | = 0 | только «рублей» / «рубля» |
| `X 000 руб` шаблон | ≤ 2 | некруглые числа или «эта планка», «такая сумма» |
| `127-ФЗ` / `ФЗ-127` | ≤ 2 | «закон о банкротстве», «федеральный закон» |
| `213.28` / `213.30` суммарно | ≤ 3 | «правило о добросовестности», «норма о завершении» |
| `ГПК` (всё) | ≤ 3 | «процессуальный кодекс», «кодекс» |
| `ГК` (всё) | ≤ 3 | «гражданский кодекс», «кодекс» |
| `000` (как часть числа) | ≤ 8 | использовать некруглые числа, «полмиллиона», «триста тысяч» |
| `ООО` в авторском тексте | ≤ 3 | «компания», «организация», «юрлицо» |

## ⚠️ Перифразы берём целыми словами, НЕ склеиваем через дефис

Лестница перифраз даёт **взаимозаменяемые слова**, а не уточнения через дефис. Каждое вхождение гипертемы заменяется на ОДНО слово из лестницы.

❌ ЗАПРЕЩЕНО склеивать роль с уточнением через дефис:
- «кредитор-поставщик», «кредитор-организация», «кредитор-юридическое лицо»
- «организация-должник», «компания-должник», «должник-гражданин»
- «Покупатель-банкрот», «фирма-банкрот», «контрагент-взыскатель»

text.ru метит такие новообразования как **ошибки правописания** (красным). Это не синонимизация, а порча текста. Если нужно показать, что должник - это компания, пиши в соседних предложениях по-разному: один раз «компания», другой раз «должник», третий «организация» - но НЕ «компания-должник» одним словом через дефис.

✅ Правильно: «Должник подал заявление. Компания просрочила платежи. Эта организация уже год не платит поставщикам».
❌ Неправильно: «Компания-должник подал заявление».

## Как использовать в writer'е

В системном промпте writer'а **обязательный шаг 0.5** (перед началом писания):

```
Прочитай meta.json → category. 
Открой соответствующий блок выше. 
Выпиши на бумагу: гипертему категории, её cap, 4-5 перифразов из лестницы.
Дальше пиши, держа эти 4-5 перифразов в активной памяти — каждое 2-е, 3-е, 4-е вхождение гипертемы берётся из лестницы по порядку.
```

После Pass A (черновик) — обязательный grep по гипертеме категории. Если она превышает cap на 30% — переписать самые «жирные» абзацы (обычно середина статьи).
