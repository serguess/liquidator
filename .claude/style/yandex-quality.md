# Требования Яндекса к качеству текста

Источники:
- https://yandex.ru/support/webmaster/ru/threat/useless-content (бесполезный контент)
- https://webmaster.yandex.ru/blog/baden-baden-novyy-algoritm-opredeleniya-pereoptimizirovannykh-tekstov (Баден-Баден)

**Главный принцип:** Яндекс понижает в выдаче страницы, которые написаны для алгоритмов, а не для людей. В YMYL-тематике (банкротство = деньги/право) санкции жёстче: статья должна реально решать задачу читателя.

## Угроза 1: Малополезный контент (useless-content)

Яндекс ищет признаки автогенерации без редактуры:

1. **Скопированный или переписанный контент.** Замена слов на синонимы первоисточника. Объединение информации с нескольких страниц в одну без добавленной ценности.
2. **Автогенерация без редактуры.** Тексты от нейросети без правок. Признаки:
   - Отклонение от темы по ходу статьи
   - Циклическое повторение одной мысли
   - Увеличение объёма ради объёма (лишние абзацы без новой информации)
   - Поверхностные рассуждения без практических деталей
   - Вымышленные факты, которые не подтверждаются источниками
3. **Автогенерируемые каталоги.** Каталоги, статьи-агрегаторы без обработки.

**Главный критерий:** «Контент не несёт дополнительную ценность пользователю и не решает его задачу».

### Как мы защищаемся

- Каждый раздел статьи должен отвечать на конкретный вопрос пользователя. Если раздел нельзя сформулировать как вопрос - его не должно быть.
- Запрещены шаблонные разделы «Проблемы и перспективы», «Заключение с оптимизмом», «Что нас ждёт в будущем».
- Каждое утверждение со ссылкой на закон проверяется в research.json. Без подтверждения = удалить.
- Объём статьи диктуется темой, а не KPI. Тема на 8000 знаков - 8000, не растягиваем до 15000 водой.
- Конкретика: цифры, статьи закона с номерами, даты, суммы, сроки в днях.

## Угроза 2: Баден-Баден (переоптимизация)

Алгоритм работает с 2017 года. Понижает страницы (не удаляет из индекса), где «текст пишется не для людей, а для поисковых алгоритмов».

Признаки:
- Избыточное повторение одной ключевой фразы (плотность ≥3-4%)
- Неестественные конструкции типа «купить эхолот недорого в Москве», «банкротство физических лиц цена быстро»
- Перечисление синонимов ключа через запятую без смысловой нагрузки
- Жёсткое попадание ключа в каждый заголовок и каждый первый абзац
- Текст, который при чтении вслух звучит неестественно из-за ключей

Учитываются поведенческие факторы (время на странице, дочитывания, отказы). Если читатели уходят со страницы быстро - это сигнал переоптимизации.

### Как мы защищаемся

**Density главного ключа: ≤2.5%.** Считаем точные вхождения + 1-2 точных вхождения частей ключа на 1000 знаков.

**Точное вхождение главного ключа:** 1 раз в первом абзаце (или title), 1 раз в H1, 1-2 раза в подзаголовках, 2-3 раза в теле. Всё. Остальное - через части ключа, синонимы, связные конструкции.

**Запрещены:**
- Конструкции «купить/заказать/оформить X в [город] недорого/срочно/быстро»
- Перечисление через запятую: «банкротство физлиц, банкротство юрлиц, списание долгов, взыскание задолженности» (если только это не уместный список услуг)
- Ключ через каждое предложение
- Title и Description, которые звучат как реклама-спам

**Обязательно:**
- Текст читается вслух как естественная речь
- Ключи распределены по тексту равномерно, а не сгруппированы
- Заголовки H2/H3 - вопросы или утверждения от лица читателя, а не SEO-конструкции

## Угроза 3: Заспамленность text.ru (целевой ≤50%)

text.ru считает не density главного ключа, а общий «спам-индекс»: долю частотных слов, повторяющихся n-граммов, плотность тематической лексики. Это отдельная метрика от Баден-Баден density.

Целевой **≤ 50%**.

Hard-блок text.ru-спама в `tools/quality_gate.py` НЕ установлен (text.ru API
у нас не подключён, метрика только для отчёта в meta.json). Контроль превентивный
через локальные пороги (`top10_share`, `ngram3_repeat_share`, `lexical_diversity`
в `tools/quality_checks.py`).

### Откуда берётся высокий индекс

1. Одно и то же существительное повторяется в каждом абзаце («банкротство», «должник», «кредитор», «процедура»). text.ru считает это спамом, даже если ключ употреблён всего 5 раз.
2. Однотипные обороты: «в случае…», «при этом…», «таким образом…» - повторение шаблонных конструкций.
3. Тематические термины кучкой: «процедура банкротства», «сумма долга», «имущество должника» - три словосочетания в одном абзаце с пересекающейся лексикой.
4. Низкое словарное разнообразие: на 7000 знаков должно быть не меньше 950
   уникальных лемм (под цель ≤50%). При <900 текст «пустой» с точки зрения text.ru,
   заспам растёт выше 50%.

### Что делать писателю

- **Чередовать существительные с местоимениями.** Не «должник подаёт заявление, должник прикладывает документы, должник ждёт решения». Лучше: «Должник подаёт заявление, прикладывает документы, ждёт решения».
- **Не повторять одно существительное больше 3-4 раз на 1000 знаков.** Если приходится назвать сущность чаще - значит, абзац разросся, разбить.
- **Разрывать тематические связки.** Если в абзаце уже есть «процедура» и «должник», следующий абзац начать с другого фокуса (даты, цифры, конкретного действия).
- **Шаблонные связки минимизировать.** «Кроме того», «при этом», «таким образом», «следует отметить», «важно понимать» - вычищать. Без них текст плотнее по смыслу и легче по индексу заспамленности.

### Что делать редактору (агент 5)

Эвристическая оценка до отправки в text.ru API:
1. Извлечь все слова длиннее 3 символов, привести к нижнему регистру.
2. Топ-10 самых частотных слов: суммарная доля от всех слов >18% = риск превышения 50% по text.ru.
3. Доля 3-граммов, встречающихся 2+ раза: >4% от всех 3-граммов = риск.
4. Уникальных лемм / общее количество слов (lexical_diversity):
   < 0.62 = низкое разнообразие, заспам text.ru > 50% (hard-блок в quality_gate);
   0.62-0.65 = граница целевой, заспам 45-50%;
   ≥ 0.65 = безопасно для цели ≤ 50%;
   ≥ 0.70 = с запасом (заспам ~40%).

Если хотя бы 2 из 3 порогов превышены - возврат на писателя с указанием «снизить плотность повторов».

## Финальный чек перед публикацией

- [ ] Каждый раздел отвечает на конкретный вопрос пользователя
- [ ] Объём диктуется темой, не растянут водой
- [ ] Все цифры/даты/статьи закона подтверждены источником
- [ ] Density главного ключа ≤2.5%
- [ ] Точное вхождение главного ключа: 5-7 раз максимум на статью
- [ ] Заспамленность text.ru ≤ 50% (выше 50% - возврат на писателя)
- [ ] Уникальность text.ru ≥ 85% (стабильно)
- [ ] Топ-10 частотных слов: суммарная доля ≤10.5%
- [ ] Уникальных лемм / общее число слов (lexical_diversity) ≥ 0.62 (hard, ≥0.65 целевой под ≤50% спама)
- [ ] Нет конструкций «купить/заказать X в Y» в авторском тексте
- [ ] Текст читается вслух как живая речь, без спотыканий на ключах
- [ ] Нет шаблонных разделов «Проблемы и перспективы», «Что ждёт нас в будущем»
- [ ] Финал - конкретное действие/совет, а не позитивный шаблон
