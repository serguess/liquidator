# Anti-Spam Red Flags - конкретные примеры для вычисления и замены

**Последовательность использования:** сначала anti-spam-playbook (§1-8), потом этот файл для детализации.

## Категория 1. Дословные цитаты закона 127-ФЗ (главный убийца уникальности)

### КРАСНЫЙ ФЛАГ: ст. 213.28 / 213.30 закона 127-ФЗ

Эти статьи дословно повторяются в 30+ конкурирующих текстах. Any exact phrase → убить.

**❌ Неправильно (дословная цитата):**
- «несостоятельность гражданина признаётся судом»
- «признание несостоятельности гражданина» (из ст. 213.28)
- «гражданин может подать заявление в арбитражный суд» (дословно)

**✅ Правильно (переписать структуру):**
- Вместо прямой цитаты → кейс: «Юрист рекомендует обращение в суд, если долг превышает сумму X»
- Вместо «признание несостоятельности» → действие: «когда судья объявляет гражданина банкротом»
- Вместо шаблонной фразы → числа: «за 6 месяцев 2026 года в Москве подано 1542 заявления»

### Техника проверки:
1. Перед сохранением `draft.md` копируй абзац в text.ru
2. Если совпадение > 50% текста целиком → перепиши абзац через кейс или числа
3. Не подменяй слова автоматическим синонимизатором

## Категория 2. Шаблонные юридические формулы (категории 11-12 из anti-template-phrases.md)

### КРАСНЫЙ ФЛАГ: Повтор официальных оборотов

**❌ Неправильно:**
- «в соответствии с требованиями закона»
- «в установленном порядке»
- «при наличии оснований»
- «в целях надлежащего исполнения»
- «согласно требованиям»
- «в связи с тем что»
- «следует отметить, что»
- «таким образом»
- «более того» (в начале абзаца)

**✅ Правильно (замены):**
- вместо «в соответствии с требованиями закона» → просто факт (убрать вводку)
- вместо «в установленном порядке» → указать конкретный орган или срок
- вместо «при наличии оснований» → описать конкретные основания
- вместо «в целях надлежащего исполнения» → переписать активным залогом
- вместо «согласно требованиям» → источник конкретно: «банк требует», «суд должен»
- вместо «в связи с тем что» → замени на «потому что»
- вместо «следует отметить, что» → удаляй целиком, начинай сразу с факта
- вместо «таким образом» → удали или замени на номер кейса
- вместо «более того» в начале абзаца → объедини с предыдущим через запятую

### Техника проверки в draft.md:
```bash
grep -E "в соответствии|в установленном|при наличии|в целях|согласно требованиям|в связи с тем|следует отметить|таким образом|более того" drafts/{slug}/draft.md
```

## Категория 3. Повторяющиеся конструкции в одном абзаце

### КРАСНЫЙ ФЛАГ: одна и та же фраза-скелет дважды

**❌ Неправильно:**
- «Заявитель приносит заявление. Заявитель прикладывает документы. Заявитель ждёт решения.»
- «Суд проверит документы. Суд одобрит или отклонит. Суд отправит уведомление.»
- «Долг возникает при просрочке. Долг растёт с процентами. Долг становится требованием.»

**✅ Правильно (разнообразие):**
- «Заявитель приносит в арбитраж документы (паспорт, СНИЛС). К пакету - полный список кредиторов и опись имущества. Рассмотрение займёт 5 рабочих дней.»
- «Судья сначала проверит полноту документов. Если всё в порядке - одобрит заявление за 1 день. После - отправит уведомления всем кредиторам.»
- «Просрочка по кредиту даже на неделю уже включает штраф. Дальше 30 дней - начисляются пени. Через 90 дней кредитор имеет право передать долг коллекторам.»

### Техника проверки:
- Прочитать абзац вслух, если слышишь одно и то же окончание 2+ раза подряд → флаг
- Использовать приём §6 из playbook: чередование коротких (5 слов) и длинных (15+ слов) предложений

## Категория 4. Перенасыщение служебными словами

### КРАСНЫЙ ФЛАГ: более 1% одного служебного слова

**Служебные слова (могут быть проблемой):**
- «должник» - максимум 10 раз на 7000 знаков
- «кредитор» - максимум 8 раз на 7000 знаков
- «суд» - максимум 10 раз на 7000 знаков
- «банкротство» - максимум 12 раз на 7000 знаков (если это гл. ключ)
- «долг» - максимум 10 раз на 7000 знаков

### Техника замены (из playbook §2, перифраз лестница):

**Вместо «должник» (максимум 30% из 10 использований = 3 раза дословно):**
1. Он / она (когда контекст однозначный) - 25%
2. Заёмщик / гражданин - 20%
3. Конкретика: «фрилансер с долгом 500 тыс.» / «бизнесмен с кредитом от Сбера» - 25%
4. Функция: «тот, кто подаёт заявление» - 10%
5. Зависимость: «человек в долгах» - 10%

**Вместо «суд»:**
1. Суд - 30% (когда необходимо, если мало)
2. Арбитраж / арбитражный суд - 25% (более конкретно)
3. Инстанция / судей / судилище (архаично, редко) - 15%
4. Судья - 15%
5. Третейское разбирательство (если уместно) - 15%

## Категория 5. Автоматические синонимизаторы (ЗАПРЕТ)

### КРАСНЫЙ ФЛАГ: видно, что текст обработан автоматически

**❌ Запрещённые признаки:**
- Странные синонимы: «явление задолженности» вместо «долг возник»
- Замена одного слова в фразе: «при возникновении оснований» вместо «при наличии оснований» (видно как подмена)
- Все слова заменены, но структура осталась: «Заявитель подносит документы. Суд проводит проверку.» (видно шаблон)
- Архаичные синонимы для современного контекста: «супруг» вместо «муж», «отпрыск» вместо «ребёнок»

**✅ Правильно (ручная переписка):**
- Переписать предложение целиком, а не подменить слова
- Менять не слова, а логику: вместо «долг возник» → «в марте пришло требование на 300 тыс.»
- Проверять читаемость: если кажется странно для живого человека → странно для text.ru тоже

## Категория 6. Пустые наполнители и лишние переходы

### КРАСНЫЙ ФЛАГ: метатекст вместо информации

**❌ Неправильно (убирать целиком):**
- «Дальше разберём...» (H2 уже сообщает что будет)
- «Если вы дошли сюда, значит...» (в юриспруденции хороший знак)
- «Правильный вопрос звучит так...» (мета-комментарий)
- «Закон даёт три способа...» → заменить на факт: «Способов три»
- «Важно помнить, что...» (лишний пафос)
- «Кстати, стоит отметить...» (устная речь, не письмо)

**✅ Правильно:**
- Начинать абзац сразу с факта, без введения
- Убирать оценочные суждения («интересно что...», «стоит помнить...»)
- Начинать с глагола или существительного: «Заявление подаётся в арбитраж в течение...» вместо «Теперь рассмотрим процесс подачи...»

## Категория 7. Декомпозиция через факты (главный приём улучшения)

Один раз на 1000 знаков минимум нужна конкретика вместо абстракции.

### КРАСНЫЙ ФЛАГ: абстракция повторяется

**❌ Неправильно:**
- «Долг можно списать через банкротство. Долг возникает при просрочке. Долг растёт быстро.»
- «Должник подаёт заявление. Должник прикладывает документы. Должник ждёт решения.»
- «Процедура занимает значительное время.»

**✅ Правильно (факты вместо повторов):**
- «Долг можно списать через банкротство. Просрочка по карте Сбера за 90 дней превращается в требование коллекторов. По статистике 60% обращений - кредиты от 300 тыс. рублей.» (номер 300 тыс. → уникальная последовательность слов)
- «Заявитель приносит в арбитраж заявление. К нему - паспорт, СНИЛС, список кредиторов, опись имущества. Решение приходит за 5 рабочих дней.» (сроки как факты)
- «6 месяцев, иногда 8.» (конкретный срок)

### Техника вставки фактов:
1. Каждый H2-раздел → минимум 1 число, дата или конкретный кейс
2. Числа берутся из `research.json` (там лежат факты)
3. Не выдумывать, только из исследования
4. Факты равномерно по тексту, не в конце

## Практический алгоритм перед сохранением draft.md

1. **Поиск (30 сек):** grep дословной цитаты ст. 213.28 / 213.30
   ```bash
   grep -i "признаётся судом\|признание несостоятельности\|в соответствии" drafts/{slug}/draft.md
   ```
   Если нашли - переписать через кейс / факт / активный залог

2. **Поиск шаблонных фраз (1 мин):**
   ```bash
   grep -E "следует отметить|в целях|при наличии|согласно требованиям|таким образом" drafts/{slug}/draft.md
   ```
   Заменить на конкретику или удалить

3. **Проверка служебных слов (2 мин):**
   ```bash
   grep -io "должник\|суд\|банкротство" drafts/{slug}/draft.md | wc -l
   ```
   Если одно слово > своего лимита - применить лестницу замен из Категории 4

4. **Проверка абзацев (3 мин):** прочитать вслух каждый абзац → если слышишь одинаковое окончание, переписать через разнообразие

5. **Финальная команда:**
   ```bash
   python -m tools.quality_checks drafts/{slug}/draft.md
   ```
   Если любой risk_flag → вернуться к шагам 1-4
